# Me Corp 2.0 — 架构与进化蓝图 (Evolution Blueprint)

> **设计语境：构建一座围绕你运转的“个人数字公司”**
>
> 融合：Stanford Smallville（沉浸可视化） × CrewAI（多智能体编排） × Edict（分权治理机制）

---

## 1. 核心产品愿景 (The Vision)

Me Corp 2.0 打破了传统记录型 App（记账、待办、打卡）的功能堆砌模型。它被设计为一个**「多智能体驱动的个人治理系统」**。

在这个系统中，你是 CEO。底层的各个功能模块（财务、行政、运营等）不再是被动的数据表，而是被赋予了专属人格、系统权限和局部记忆的 **AI 高管（C-Suite Agents）**。
它们每天在你的虚拟公司里互相协作、互相扯皮、互相制衡，最终目标只有一个：**提升你的生活质量与执行效能。**

---

## 2. 核心架构：14 个智能体角色 (Agent Topology)

借鉴现代企业治理和 Edict 框架的思路，系统被划分为 **辅助（2）+ 决策治理（3）+ 业务执行（9）** 共计 14 个独立 Agent。
每个 Agent 都拥有独立的上下文、独立的工具调用权限（Tool Use）和可独立配置的 LLM 模型。

### 2.1 角色定义与分工

| 层级 | Agent 代号 | 现代公司角色 | 核心职责与设定 |
|---|---|---|---|
| **辅助** | `triage` | **Triage Bot (前台接待)** | 消息总分拣。识别闲聊直接回复（省 Token）；识别简单指令走快速查询通道；识别复杂长远指令，创建 `MC-YYYYMMDD-XXX` 正式任务单进入流水线。 |
| **辅助** | `briefing` | **Morning Brief (晨会专员)** | 每天固定的 Cron Job。整合各部门昨夜的数据回溯和外部订阅资讯，生成图文并茂的《CEO 每日早报》。 |
| **治理** | `strategy` | **Strategy Office (战略规划办)** | 接收正式任务，负责「拆解」。将一句复杂指令拆分为多个具体的子步骤，并指派给对应的业务执行部门。 |
| **治理** | `review` | **Review Board (审核委员会)** | **核心风控**。接收战略办的拆解方案进行多维校验（预算是否充足？健康是否允许？计划是否冲突？）。若任何一项不满足要求，直接 `REJECT` 驳回要求重做。 |
| **治理** | `dispatch` | **Operations Hub (运营调度中心)** | 审核通过后的「发令枪」。负责将拆解的子任务并行派发给九大业务部门，追踪执行进度，并在所有依赖项完成后汇总结果向 CEO 汇报。 |
| **执行** | `cfo` | **CFO (财务总监)** | 【账户、记账、预算、期权、订阅】—— 掌管公司的经济命脉。 |
| **执行** | `coo` | **COO (运营总监)** | 【任务看板、SOP 习惯打卡、OKR】—— 每天挥着鞭子逼你推进目标的残酷教练。 |
| **执行** | `chro` | **CHRO (人力总监)** | 【技能树、培训、身心关怀、绩效】—— 追踪你的成长轨迹，记录昨晚的睡眠情绪打卡。 |
| **执行** | `cpo` | **CPO (公关总监)** | 【人脉档案、客情维系、送礼】—— 你的社交大脑，永远记得重要人物的生日忌口。 |
| **执行** | `cao` | **CAO (行政总监)** | 【物资库存、固定资产、采购审批、证照】—— 管理你的物理资源。买任何贵重物品都需要过他这一关。 |
| **执行** | `cwo` | **CWO (健康中心总监)** | 【体重、膳食、运动、体检】—— 守护你的身体指标。 |
| **执行** | `cto` | **CTO (研发总监)** | 【项目工坊、创意池、技术沉淀】—— 管理个人的 Side Project 开发与技术笔记。 |
| **执行** | `cso` | **CSO (商务总监)** | 【收入流、商务合作、搞钱】—— 你的外部搞钱参谋，管理你的职场和副业渠道。 |
| **执行** | `clo` | **CLO (法务总监)** | 【合同、保险、纠纷追踪】—— 管理你的风险敞口和法律契约。 |

### 2.2 权限矩阵与通信规则 (Permission Matrix)

**铁律 1**：Agent 之间不可越级通信，必须遵守白纸黑字的权限表。系统底层会在路由层硬拦截。
**铁律 2**：规划者不审核，审核者不执行，执行者不规划。

- **Strategy** 绝不能绕过 **Review** 直接向底下的业务线发令。
- 业务线（如 CFO 和 CAO）相互之间若需要数据交换，必须通过 **Dispatch (Ops Hub)** 中转。
- 只有 **CEO、Triage 和 Dispatch** 能直接给你（用户）发消息。

## 3. 标准任务状态机 (Workflow Lifecycle)

当你在输入框打下一句：“下周三是我妈生日，预算 2000，帮我挑个礼物，另外提醒我最近少喝点咖啡。”
这句自然语言如何在系统内流转？

```mermaid
graph TD
    A[CEO Input] --> B(Triage Bot)
    B -- 判断为"正式任务" --> C[Strategy Office 拆解方案]
    
    C -->|方案：1.CPO准备送礼计划 2.CFO核准预算 3.COO加喝水习惯| D{Review Board 审议}
    D -- 发现预算本月已超标 --> E[REJECT 驳回重拆]
    E --> C
    
    D -- 检查通过 / 或 CEO 强制 Override --> F[Ops Hub 调度]
    
    F -->| 并发执行 | G1[CPO: 更新客情系统]
    F -->| 并发执行 | G2[CFO: 记账扣除期权积分]
    F -->| 并发执行 | G3[COO: 新增习惯 SOP]
    
    G1 --> H[Ops Hub 汇总回调]
    G2 --> H
    G3 --> H
    
    H --> I((向 CEO 回报))
```

所有任务生成独一无二的编号（如 `MC-20261012-001`），任何时候都能查看这个任务在哪个节点，产生了什么思考，调了什么 SQL。

## 4. 可观测性与容错引擎 (Observability & Scheduler)

照搬并优化 Edict v2 的工业级容错思想，治愈多 Agent 系统“死循环”、“信息黑洞”的痛点。

### 4.1. 四阶段调度器 (Robust Scheduler)
Agent 调用 LLM 时偶尔会 JSON 错误，或调外部 API 失败。系统内置挂起/恢复机制：
1. **自动重试 (Auto-Retry)**：带指数退避的重试（最大勉强 3 次）。
2. **升级处理 (Escalate)**：3 次仍失败？丢出告警，任务转黄灯（`🟡 Blocked`），交予 Ops Hub 协调。
3. **回滚 (Rollback)**：对于涉及财务数据写入的严重错误，若无法推进，执行数据库的 Transaction Rollback 并取消任务。
4. **手动介入 (Manual Intervention)**：你在后台可以随时点一个红色的“中止”或“恢复”按钮。

### 4.2. Agent 思维可视化 (Transparent Thinking)
在 Dashboard 的“指挥中心”，你可以随时点开一个任务卡片。里面清晰展示树状结构的思考日志：
> ▾ **CFO (Ada) 的思考链路**
> 1. 收到指令：检查 [数码设备] 本月余额。
> 2. `Tool Call`: `select_budget_by_category('数码')` -> 返回 1200 元
> 3. `Thinking`: 目标预算是 3500 元，当前可用只有 1200。差距过大。无法同意。
> 4. `Output`: 返回给 Review Board -> REJECT 意见。

## 5. UI/UX 沉浸式场景 (Virtual Office UI)

这是系统的感性骨架。不堆砌报表，我们要让你感觉在“玩游戏”。

### 5.1 公司大楼视角 (The Main Hub)
采用 **PixiJS (React)** 渲染一个微型等距视图（Isometric）现代公司大楼。
- 每个楼层是一个部门，点进去进入具体的看板。
- 代表各个 Agent 的像素小人在楼层里活动。
  - **工作态 (Working)**：疯狂敲着极客键盘。
  - **空闲态 (Idle)**：喝着咖啡摸鱼。
  - **驳回态 (Blocked)**：抓头、叹气、焦急踱步。
  - **通信态 (Talking)**：楼层之间会有高亮的激光连线闪烁，表示 Agent 正在异步讨论。

### 5.2 每日站会气泡 (Morning Standup)
结合 **Framer Motion** 的弹性动画。每天你第一次打开界面，不用你自己去看数据，而是各个高管以社交软件对话框的方式给你弹气泡做“述职”。
> *“Boss，昨日你的生产力大幅下降啊（COO）”*
> *“是的，而且他昨晚外卖点得太贵了，预算紧迫（CFO）”*

## 6. 用户自定义系统 (Persona Studio)

一切都可以按你的口味来。系统内置全局的控制面板：

1. **设定你自己**
   - 称呼设定：让一切高管叫你 “Boss”、“主公”、“布鲁斯韦恩”都可以。
   - 特权策略：如果你不想频繁被打扰，可以开启「民主模式」，也可以开启「随心所欲独裁模式」（但这会扣除期权积分）。

2. **捏脸/塑造高管 (Agent Config)**
   - 对于系统内所有的 14 个 Agent，均可编辑对应的 JSON 配置文件。
   - **名称与头像**：把 CFO 叫 “老王 🧙‍♂️” 或者 “Ada 👩‍💼”。
   - **人设 Prompt**：控制它的发言Tone调。比如把 COO 设定为“严厉毒舌的魔鬼健身教练”，它审核你买零食的申请时就会极尽嘲讽之能事。
   - **底层模型切换**：比如让需要严密长逻辑的 Strategy 办公室接 Claude 3.5 Sonnet，让负责高频杂务闲聊的 Triage 接 DeepSeek V3，让写代码的 CTO 接 GPT-4o。

## 7. 游戏化成长体系 (Gamification)

通过数据闭环把个人的苦行僧修行变成“打怪升级”。

1. **成就奖章**
   - 「Budget Master」：连续 3 个月消费未触顶。
   - 「Early Bird」：连续 7 天 23点前入睡。
   - 「暴君特权」：单月使用 Override 强行越过审核委员 10 次以上获得（负面属性成就）。

2. **CEO 评级指数 (Performance Rating)**
   根据你在“财务健康”、“规律作息”、“OKR达成”、“人脉活跃度”等硬性指标数据的加权，你的职位会逐阶上升：
   `Intern (实习生)` -> `Manager` -> `VP` -> `CEO` -> `Legendary founder (传奇创始人)`

3. **高管考绩 (Agent KPI)**
   - 每个 Agent 也会被评分！如果 Review Bot 频繁过度驳回导致任务流产，或者 CFO 生成计划消耗 Token 过多，你在面板上会看到他们的“绩效滑坡”，可以通过修改 Prompt 找他们“谈心”。

## 8. 技术栈与阶段路线图 (Stack & Roadmap)

### 前后端技术基底
- **Frontend**: React 18, Vite, Tailwind CSS, Zustand, Recharts
- **Animations**: Framer Motion (微交互气泡), PixiJS (公司大楼全景), Lottie (成就绽放动画)
- **Backend**: Node.js, Express, TypeScript (目前), 视多模态与调度需求后续可混搭 Python (LangGraph) 专门处理 Agent Workflow
- **Database**: PostgreSQL (包含 `pgvector` 插件存储个人日记的长期向量记忆)

### 阶段推进计划
- **Phase 0（基石建设 · 当前状态 ✅）**:
  - `users`, `finance`, `operations`, `hr`, `pr`, `admin` 等九大表结构通过 `init.ts` 创建完成。
  - 标准 CRUD 原型 API 调通，基础 `corp-card` React UI 板块呈现。

- **Phase 1（UI 翻新与动画装配 · Next）**:
  - 全面引入 Framer Motion 重做路由转场。
  - 开发 `Persona Studio` 界面并挂载全局配置，搭建假数据支撑的 Daily Standup 早会动效页面。

- **Phase 2（单节点智能与 API 接入）**:
  - 把 14 个 Agent 的 `System Prompt` 撰写完毕落入本地磁盘。
  - 开发工具层能力（Tools），让后台能调用现有的 Postgres 查询。
  - 跑通 `Triage` 和 `Morning Brief` 这俩轻量级单点助理的真实大模型回溯链路。

- **Phase 3（中枢调度与多轮博弈）**:
  - 实现最硬核的机制：`Strategy` -> `Review` -> `Dispatch` 仲裁流水线。
  - 后台重构基于 Agent ID 的四阶错误处理器（重试、报警、回滚），开发“指令控制看板”。
  
- **Phase 4（终极进化与长效记忆）**:
  - 研发 PixiJS 2D 办公楼主卧。
  - 引入 RAG 向量记忆库，让 CFO 记住你在三个月前因为一笔冲动消费后悔过的情绪（从你的 `wellness_logs` 里提取特征）。
